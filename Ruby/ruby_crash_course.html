A little bit of Object Orientation

Encapsulation

Separation between interface and implementation.

The idea is that you can create an interface (Public methods in a class) and, as long as that interface remains consistent, the application can interact with your objects.



Example, the algorithm used to compute pi might be proprietary. You can expose a simple API to the end user, but you hide all of the logic

Polymorphism

This is reflected in the capability to write one routine that can operate on objects from more than one class. 

# creates a new song witn the music name only
def new_song music_name
end

# creates a new song duplicating an
# existing one
def new_song song
end

Inheritance

Derive a new class from an existing one and acquire all the feature of the existing class. 

class figure
  @number_of_sides
  @name

  def initialize
    @number_of_sides = 0
    @name = ''
  end

  def get_name
  end
end

class square < figure
  def initialize
    @number_of_sides = 4
    @name = 'square'
  end
end

Classes 

#!/usr/bin/ruby

class Song
  def initialize(name = 'untitled', artist = 'unamed', duration = 0)
    @name     = name
    @artist   = artist
    @duration = duration
  end

  def to_s
    "Song: #{@name}--#{@artist} (#{@duration})"
  end
end

aSong = Song.new()
puts aSong.to_s

aSong = Song.new("My Way", "Sinatra", 225)
puts aSong.to_s

> Song: untitled--unamed (0)
> Song: My Way--Sinatra (225)

Attribute Acessors

Ruby has a built in facility to produce getters and setters for us.

The programmer must simply write attr_accessor :var_name, it then creates at runtime the following code: 

attr_accessor :age

def age=(value)
  @age = value
end

def age
  @age
end

You can also write attr_reader and attr_writer to create attributes that can be only written or read-only
